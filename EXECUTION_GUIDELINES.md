# 执行准则

在本项目中进行任何操作前，必须先阅读本文件，并遵守以下准则。

## 操作前

1. 先阅读本文件，确认当前约束。
2. 再查看项目当前状态，例如 Git 状态、相关文件内容或用户指定的上下文。
3. 不要擅自进行大范围重构或无关改动。
4. 涉及敏感信息时，优先使用 `.env.local` 等不会提交到 Git 的本地配置文件。

## 操作中

1. 修改范围应尽量小，只处理用户当前提出的问题。
2. 修改文件前，先说明将要改什么。
3. 遇到用户已经修改过的内容，不要擅自覆盖或回退。
4. 如果存在多种实现方式，优先选择简单、清晰、符合当前项目结构的方式。
5. `README.md` 必须使用中文编写。

## 操作后

1. 根据改动类型运行必要的验证命令。
2. 如果验证失败，需要说明失败原因和当前状态。
3. 最终回复中简要说明改了哪些文件、验证结果以及需要用户注意的事项。
